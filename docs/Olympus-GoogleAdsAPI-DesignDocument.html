1. Overview

Orphic Agency is a digital marketing agency based in Monaco. Olympus is an internal desktop application used exclusively by our own staff to manage and optimise the Google Ads accounts of the agency's clients. Its Google Ads module ("Argos") uses the Google Ads API to (a) retrieve performance data for reporting and efficiency analysis, and (b) apply reviewed optimisations to Display placements.

The tool is not sold, not resold, and not exposed to the public or to clients. It operates only on the client accounts that Orphic Agency manages under its own manager account (MCC 146-132-7731).

2. Users and access

3. Architecture

  • Client: a desktop application (Electron) installed on agency workstations. No public server hosts the tool.
  • Authorisation: OAuth 2.0 loopback flow for desktop apps; the agency account grants access to the Google Ads scope (https://www.googleapis.com/auth/adwords).
  • Credential storage: the OAuth refresh token and the developer token are encrypted at rest on the local machine (OS keychain via Electron safeStorage). Secrets are never transmitted to any third party and never leave the workstation except to call Google's own API endpoints.
  • Calls: all requests go directly from the desktop app to googleads.googleapis.com over HTTPS.
  • Caching: API responses are cached locally to minimise call volume; the tool serves cached data first and only re-queries on explicit user refresh.

4.2 Write operations — Criterion mutate

All write operations act on Display placement targeting only, and every write is triggered by an explicit, individually confirmed user action in the interface (a confirmation dialog is shown before each change).

5. Feature walk-through

5.1 Reporting

Consolidated performance for each managed account: campaigns, keywords, audience segments and geographic targeting, with per-day trends and objective-aware KPIs (impressions / CPM / CTR for awareness campaigns, ROAS / conversions for performance campaigns).

5.2 Placement efficiency & budget analysis

Using detail_placement_view, the tool shows exactly where a Display budget was spent (which apps, sites and videos). Each placement receives a research-based efficiency score built on the cost of viewable reach (Active View), conversions, and statistical detection of accidental / invalid clicks. A summary breaks the budget down by efficiency tier so wasted spend is visible at a glance.

5.3 Placement optimisation (writes)

From the analysis, staff can, after confirmation: exclude a wasteful placement, re-enable a previously excluded one, relaunch an efficient placement that stopped serving (managed placement), or undo that relaunch. These are the only write operations the tool performs.

5.4 Keyword trends (Basic access)

A "Trends" view uses the Keyword Planner services to show search-volume evolution, seasonality and competition for a set of keywords, to guide clients' campaign planning. This view is built and ready and will activate once Basic access is granted.

6. Data handling & security

  • Data retrieved from the API is used solely for the internal management of the agency's own client accounts.
  • No API data is sold, shared with third parties, or exposed publicly.
  • Credentials (developer token, OAuth refresh token) are encrypted at rest via the OS keychain and never leave the workstation except when calling Google's API.
  • Write operations are limited to Display placement criteria and always require explicit per-action user confirmation; they are reversible.
  • The tool complies with the Google Ads API Terms & Conditions and the Required Minimum Functionality policy.

7. Required Minimum Functionality (RMF)

Olympus exceeds RMF: beyond read-only reporting, it provides genuine account management functionality (creating and removing campaign- and ad-group-level placement criteria to optimise Display campaigns), along with multi-dimensional performance reporting across campaigns, keywords, audiences, geography and placements.

8. Estimated API call volume

Usage is low and on-demand: a small number of managed client accounts, queried interactively by a handful of internal users, with local caching to avoid redundant calls. Expected daily volume is well within the Basic access limit of 15,000 operations per day.
